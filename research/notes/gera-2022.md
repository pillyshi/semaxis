## Why It Matters For SemAxis

Gera et al. show that entailment-based zero-shot classifiers are not merely
fixed off-the-shelf baselines. They can be adapted to a target task using only
class names and unlabeled in-domain examples, by fine-tuning on the model's
most confident pseudo-labels.

For SemAxis, this is relevant in two ways. First, a direct zero-shot NLI
baseline in the label-proxy leakage audit could reasonably have an adapted
variant, as long as adaptation is fold-local and uses only allowed unlabeled
training-fold text. Second, their results reinforce that NLI behavior depends
on exposure to the specific class names and hypothesis template. SemAxis
should therefore log exact hypothesis wording, template wording, NLI backbone,
fold, seed, and any adaptation procedure.

This paper does not directly compete with SemAxis's feature-matrix claim. It
adapts class-label NLI prediction, whereas SemAxis induces many explanatory
hypothesis features for downstream estimators.

## Method

The paper starts from an entailment-based zero-shot classifier. For each text
and class, a hypothesis is built with the Hugging Face default template
`This example is [].`, and the class with the highest entailment score is the
prediction.

Given an unlabeled set `U`, the method constructs pseudo-labeled NLI training
pairs:

1. For each class, select examples where that class has the highest entailment
   score.
2. Rank those examples by the margin between the best and second-best class.
3. Keep the most confident examples as positive entailment pairs.
4. Add contradiction pairs by replacing the positive class with another class.
5. Fine-tune the original NLI model on these pseudo-labeled pairs.
6. Repeat the pseudo-labeling and fine-tuning process for additional
   iterations.

The paper compares four negative-example strategies: random class replacement,
second-best class replacement, furthest class replacement, and all other
classes. It also masks the token most similar to the class name using GloVe
similarity, making selected confident examples less trivial.

## Evaluation

- Datasets: 20 Newsgroups, AG News, Amazon reviews, DBPedia, GoEmotions, IMDB,
  ISEAR, and Yahoo Answers.
- NLI backbones: `roberta-large-mnli`, `deberta-large-mnli-zero-cls`, and
  `bart-large-mnli`.
- Unlabeled data: up to `10,000` training examples per dataset; labels are not
  used.
- Repetitions: five random seeds.
- Fine-tuning: one epoch, learning rate `2e-5`, batch size `32`, AdamW, single
  NVIDIA A100.
- Main metric: test accuracy after two self-training iterations.

The reported average test accuracy improves for all three backbones:

- BART: `61.9` to `71.7`
- DeBERTa: `63.5` to `72.8`
- RoBERTa: `57.4` to `70.9`

The gains are largest on topic-style datasets such as 20 Newsgroups, DBPedia,
and Yahoo Answers, and smaller on already-strong binary sentiment tasks such
as Amazon and IMDB.

## Implications For SemAxis

Self-training adapts the NLI model to the class names and hypothesis template.
That is a useful reminder for SemAxis: feature scores are not purely intrinsic
properties of the text and hypothesis string. They also depend on the NLI
model's adaptation history, the template, and the available unlabeled corpus.

In a leakage audit, a self-trained direct NLI classifier could be an informative
upper or stress-test baseline: if SemAxis's supervised generated features
collapse to label paraphrases, then adapted direct NLI may be a simpler
competitor. However, this should not be folded into the first audit unless the
scope allows careful fold-local adaptation.

The method also raises data-use concerns. Using unlabeled examples from the
test fold would contaminate evaluation if the goal is a supervised train/test
protocol. Any SemAxis comparison using self-training must specify whether it is
transductive, train-fold-only, or uses a separate unlabeled pool.

## Comparison With Current SemAxis

| Dimension | Gera et al. | SemAxis Today |
|---|---|---|
| Goal | Improve direct zero-shot NLI classification | Generate NLI-scored feature matrix |
| Hypotheses | Class-label template hypotheses | LLM-generated explanatory hypotheses |
| Adaptation | Fine-tune NLI model on pseudo-labeled unlabeled data | No NLI adaptation |
| Data required | Class names and unlabeled target-task texts | Texts plus optional labels for generation |
| Output | Class prediction | Feature matrix for downstream estimator |
| Main risk | Transductive use and pseudo-label noise | Label-proxy features, redundancy, NLI measurement reliability |

## Actionable Findings

1. Keep direct zero-shot NLI in the label-proxy leakage audit, and consider an
   adapted self-training variant only as an explicitly scoped extension.
2. If self-training is evaluated, require fold-local or clearly transductive
   protocol labeling. Do not silently use test-fold texts as unlabeled
   adaptation data.
3. Add adaptation metadata to experiment audit logging if this baseline is ever
   implemented: unlabeled pool identity, pseudo-label selection rule, iteration
   count, negative-example strategy, NLI backbone, and template wording.
4. Use this paper as evidence that sample selection is not only an LLM
   prompting concern; confident or diverse example selection also matters for
   NLI adaptation.
