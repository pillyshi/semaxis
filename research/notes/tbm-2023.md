## Why It Matters For SemAxis

Text Bottleneck Models (TBM) are a direct conceptual competitor. TBM
automatically generates a sparse set of human-readable concepts, measures each
concept for every text, and gives only those measurements to a linear
predictor. This overlaps with SemAxis's intended value proposition more deeply
than a baseline that merely asks an LLM to classify text.

The central distinction is measurement. TBM uses an LLM to answer
concept-specific questions and map responses to scores. SemAxis represents
features as declarative hypotheses and scores them with an NLI cross-encoder.
SemAxis must show that this replacement improves cost, repeatability, or
evaluation safety rather than claiming the bottleneck idea itself is new.

## Method

TBM consists of three modules:

1. Concept Generation starts with an empty concept set and iteratively proposes
   a new concept using examples poorly represented or misclassified by the
   current concept-space predictor.
2. Concept Measurement prompts an LLM zero-shot with each concept's question,
   response guide, and mapping to obtain a numerical concept score and
   supporting snippet for an input text.
3. Prediction Layer trains linear regression or logistic regression using only
   the measured concept scores.

A generated concept is admitted only if adding it improves a provisional TBM
on a subset of the training data by a threshold. Previously generated concepts
are included in generation prompts to discourage duplicates.

## Evaluation

- Data: 12 datasets covering sentiment, natural language inference, topic
  classification, scientific citation intent, fake news, and political
  partisanship.
- Typical budget: 250 training and 250 test examples due to LLM query cost.
- Baselines: fine-tuned BERT-base, fine-tuned DeBERTa-base, TF-IDF Naive
  Bayes, 10-shot GPT-4, and fine-tuned GPT-3.5.
- Prediction metrics: accuracy for classification and MSE for regression.
- Interpretability evaluation: human review of concept generation and human
  agreement with concept measurement.

TBM performs strongly on sentiment tasks: its reported classification average
is `0.896`, close to fine-tuned GPT-3.5 (`0.900`) and above GPT-4 10-shot
(`0.840`) in the reported sentiment classification subset. Its performance is
more uneven outside sentiment: the reported non-sentiment accuracy average is
`0.745`, below fine-tuned GPT-3.5 (`0.823`) and GPT-4 10-shot (`0.765`).
The paper attributes part of this gap to tasks whose concepts do not combine
well in a simple linear bottleneck.

## Failure Modes Relevant To SemAxis

The human evaluation of generated concepts reports:

- Redundancy as the most common issue, affecting `25%` of concepts on average.
- Leakage as the second most common issue, affecting `15%` on average.
- Measurement quality varying by task: CEBaB reaches median
  machine-human correlation `0.814` and exact match `0.893`, whereas Fake News
  reaches only `0.317` and `0.549`.
- Small-data runs can improve through roughly five concepts and then degrade
  as overly specific or leaky concepts enter the bottleneck.

These are directly relevant to SemAxis. NLI assignment does not prevent a
generator from emitting duplicate, label-parroting, or otherwise misleading
hypotheses. Further, a readable hypothesis bank can still be unstable or
overfit if features are selected using insufficient training evidence.

## Comparison With Current SemAxis

| Dimension | TBM | SemAxis Today |
|---|---|---|
| Feature representation | Structured categorical concept and response guide | Declarative hypothesis string |
| Concept/feature generation | Iterative, driven by current model errors | One generation call per collection or class contrast |
| Value measurement | LLM answer mapped to numerical concept score | NLI entailment probability |
| Feature admission | Retain candidate when training-subset score improves | Retain all returned hypotheses |
| Prediction layer | Linear regression or logistic regression | Any sklearn downstream estimator |
| Known feature audit | Human-evaluated redundancy and leakage | Not yet implemented |

Relevant SemAxis code:

- `src/semaxis/supervised.py`: produces discriminative hypotheses from sampled
  class contrasts, without validation or pruning.
- `src/semaxis/unsupervised.py`: produces collection hypotheses, without
  validation or pruning.
- `src/semaxis/nli.py`: makes measurement cheaper and more constrained than
  per-feature LLM question answering, but does not validate feature quality.

## Actionable Findings

1. Promote semantic hypothesis deduplication as an issue candidate. Both FELIX
   and TBM identify redundant generated features as a concrete problem.
2. Add a separate future candidate for train-fold-only feature validation or
   pruning. TBM's candidate admission and small-data overfitting results suggest
   that deduplication alone will not address leaky or overly specific features.
3. In benchmark planning, include redundancy, label-overlap/leakage auditing,
   stability across seeds or folds, and retained-feature count alongside
   predictive performance.
4. Treat TBM as a comparison target for the research claim, even if full
   reproduction is too expensive for the initial implementation PR.

## Issue Candidate Impact

TBM strengthens the existing
